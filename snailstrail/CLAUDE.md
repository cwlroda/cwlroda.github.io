## What This Is

Snail's Trail is a daily Wikipedia navigation game (like Wordle meets WikiGame). Players hop between Wikipedia articles trying to reach a target article within 10 hops. Semantic similarity (via in-browser ML embeddings) guides the player and drives scoring. Each day generates deterministic puzzles per difficulty level.

## Build & Run

```bash
npm install
npm run dev          # Vite dev server (hot reload)
npm run build        # Production build to dist/
npm run preview      # Preview production build
npm run fetch-articles   # Re-fetch vital-articles.json from Wikipedia API
npm run deploy       # Build + deploy to GitHub Pages via gh-pages
```

No test framework is configured. No linter is configured.

## Architecture

**Stack**: React 18 + Vite 6, no router, no state library. Single-page app with game state managed in `App.jsx` via `useState`.

**Game flow**: `loading` (model download) -> `menu` (difficulty select) -> `generating` (puzzle pair selection) -> `playing` (article navigation) -> `ended` (score + share).

### Embedding Pipeline (critical path)

The app runs `Xenova/all-MiniLM-L6-v2` entirely in-browser via a Web Worker:

1. `useEmbedder` hook spawns `embedder.worker.js` on mount, loads the Transformers.js model
2. Worker exposes `EMBED` (title -> vector) and `SIMILARITY` (title pair -> cosine similarity) messages
3. Hook wraps these in promise-based `embedArticle()` and `getSimilarity()` via a pending-request map
4. Embeddings are cached in the worker's `Map` (title -> Float32Array)

All similarity is computed on **article titles only**, not article body text.

### Daily Puzzle Generation

`useDailyPuzzle` generates deterministic puzzles using date+difficulty as seed (`djb2` hash -> `mulberry32` PRNG). Pairs are sampled from `vital-articles.json` (~1000 Wikipedia Level 4 vital articles) and filtered by embedding similarity range:
- Easy: 0.5-0.9 similarity
- Medium: 0.25-0.5
- Hard: 0.0-0.25

Puzzles are cached in localStorage per date+difficulty.

### Wikipedia Integration

`useWikipedia` fetches article HTML section-by-section via the Wikipedia Parse API (`action=parse`), accumulating sections until at least 8 links are found (max 4 sections). HTML is sanitized via DOMParser with noise elements removed. Articles are cached in a ref-based Map.

### Scoring

- Per-hop: `similarity^2 * 100`, penalized 0.4x for hub articles (super-connected pages like "United States")
- Win bonus: sum of increasing bonuses for each unused hop (15 + 20 + 25 + 30... pts)
- "Show more" section expansion: 5 pts per section
- Hub lists are tiered by difficulty (harder = more articles penalized)
- Difficulty unlocks: Medium requires **winning** Easy, Hard requires **winning** Medium

### Key Data

- `src/data/vital-articles.json` — ~1000 article titles, generated by `scripts/fetch-vital-articles.js`
- `src/data/hubs.js` — tiered hub article lists (score penalty targets)
- Progress/puzzle cache stored in `localStorage` under `snailstrail_*` keys

## Conventions

- Plain `.jsx` files (no TypeScript)
- Hooks in `src/hooks/`, components in `src/components/`, utilities in `src/utils/`
- Worker communication uses `{ type, payload }` message protocol
- Vite base path is `/snails-trail/` (for GitHub Pages subdirectory deployment)
- Wikipedia titles are normalized via `decodeURIComponent(title.replace(/_/g, ' '))`
